## 1. 시작하기 전에

### 권장 환경

| 항목 | 권장 사양 |
|------|-----------|
| 브라우저 | Chrome / Edge 최신 버전 |
| 해상도 | 1280 × 720 이상 |
| JavaScript | 활성화 필수 |
| 그래픽 | WebGL 사용 가능 환경 권장 |

### 로컬 실행

```bash
git clone https://github.com/NaldonStudy/umamusmaeORDER.git
cd umamusmaeORDER
npm install
npm run dev
```

브라우저에서 `http://localhost:5173`으로 접속하면 됩니다.

## 2. 앱 실행하기

앱을 열면 먼저 `TeamSetup` 화면이 표시됩니다.  
여기서 참가 팀을 만든 뒤 레이스를 시작하면, 카운트다운 후 `RaceTrack`의 Pixi 레이스 씬으로 넘어갑니다.

```text
팀 등록
  → 레이스 시작
  → 카운트다운
  → Pixi 레이스 관전
  → 결과 화면
```

---

## 3. 팀 등록하기

### 등록 규칙

- 최소 2팀, 최대 12팀까지 참가할 수 있습니다.
- 각 팀은 고유한 이름과 대표 색상을 가져야 합니다.
- 레이스 화면의 주자 색상, 미니맵 점, 순위 오버레이는 이 대표 색상을 기준으로 표시됩니다.

### 입력 항목

| 항목 | 설명 |
|------|------|
| **팀 이름** | 레이스 HUD와 결과 화면에 표시될 이름 |
| **대표 색상** | 주자 강조색, 순위 표시점, 미니맵 표시 색상 |
| **기초 스탯** | 속도 / 가속 / 스태미나 중심의 기본 성능 |
| **스킬** | 레이스 중 조건부로 발동하는 특수 효과 |

### 수정과 삭제

- 이미 등록한 팀은 다시 선택해 수정할 수 있습니다.
- 필요 없는 팀은 삭제 후 다른 조합으로 다시 편성할 수 있습니다.

## 4. 스탯과 전략 이해하기

### 속도

- 기본 페이스를 결정합니다.
- 초반 선두 확보에 유리합니다.

### 가속

- 순간적인 치고 나감을 결정합니다.
- 추월과 스킬 폭발력에 영향을 줍니다.

### 스태미나

- 후반 유지력을 결정합니다.
- 막판 급격한 감속을 줄여줍니다.

### 컨디션

- 레이스 시작 시 자동 반영되는 보정 요소입니다.
- 매 레이스 결과가 조금씩 달라지는 이유 중 하나입니다.

### 추천 전략

| 스타일 | 추천 조합 |
|--------|-----------|
| **선행형** | 속도 중심 + 선두 유지형 스킬 |
| **추입형** | 가속 / 스태미나 중심 + 후반 발동형 스킬 |
| **균형형** | 모든 스탯을 고르게 배분해 안정성 확보 |

## 5. 스킬 선택하기

각 팀은 레이스 도중 자동 발동할 스킬을 선택합니다.  
스킬은 레이스 엔진에서 조건을 만족할 때 발동하며, Pixi 씬의 시각 효과와 로그에도 연결됩니다.

### 대표 스킬 예시

| 스킬명 | 발동 상황 | 기대 효과 |
|--------|-----------|-----------|
| **선두 질주** | 선두권 유지 | 선행 강화 |
| **막판 스퍼트** | 종반 진입 | 후반 가속 강화 |
| **추격의 불꽃** | 중하위권 | 순간 추월력 상승 |
| **랜덤 부스트** | 임의 시점 | 짧은 속도 상승 |
| **방해 저항** | 혼전 구간 | 페이스 유지 |
| **침착한 페이스** | 전반 운영 | 체력 관리 |

### 스킬이 화면에 반영되는 방식

- 발동 시 상단 또는 씬 내부에서 배너 / 플래시 효과가 나타납니다.
- 일부 버스트 계열 효과는 주자 뒤쪽의 트레일이나 속도선으로 강조됩니다.
- 추월, 결승, 스킬 발동은 로그와 화면 흔들림 효과로 체감할 수 있습니다.

## 6. Pixi 레이스 화면 이해하기

레이스를 시작하면 짧은 카운트다운 뒤 Pixi 기반 레이스 씬이 실행됩니다.  
이 구간에서는 `RaceTrack`이 틱 기반 상태를 갱신하고, `PixiRaceScene`이 트랙, 주자, 카메라, FX, 미니맵을 매 프레임 렌더링합니다.

### 현재 화면 구성

| 영역 | 설명 |
|------|------|
| **레이스 헤더** | 진행률, 완주 수, 상단 타이틀 표시 |
| **메인 트랙** | 타원형 코스 위에서 각 팀이 레인별로 주행 |
| **카메라 추적** | 선두 주자 중심으로 부드럽게 이동 |
| **실시간 순위 오버레이** | 상위 팀의 현재 순위를 우측 오버레이에 표시 |
| **오벌 미니맵** | 우측 상단에 전체 트랙 기준 위치 표시 |
| **스킬 로그** | 하단에서 주요 이벤트를 텍스트로 중계 |
| **오디오 컨트롤** | 상단에서 음소거 또는 볼륨 조절 가능 |

### 레이스 씬에서 보이는 연출

- 주자는 레인별 오프셋을 가진 타원 트랙 위를 달립니다.
- 관중석은 원경 / 중경 / 전경 레이어로 구성되어 카메라 이동 시 패럴랙스 느낌을 줍니다.
- 먼지, 플래시, 화면 흔들림이 주행 속도와 이벤트에 따라 반응합니다.
- 결승선은 트랙 위 기준선으로 표시됩니다.

### 고도화 예정 요소

현재 문서 기준으로 다음 연출은 추가 또는 강화 예정입니다.

| 예정 항목 | 설명 |
|----------|------|
| **스프라이트 시트 러너** | 단색 주자 대신 달리기 프레임 애니메이션 적용 |
| **고급 스킬 FX** | `burst`, `skill_activate`, `overtaken` 이벤트별 링 플래시, 추월 하이라이트, 강화된 후방 트레일 |
| **오디오 시스템** | 관중 함성 루프, 풍절음 레이어, 스킬 / 추월 / 결승 효과음 |
| **오디오 UI** | 상단 음소거 / 볼륨 토글 |
| **성능 최적화** | 12팀 기준 60fps 목표, 파티클 풀링과 오브젝트 재사용 |

## 7. 레이스 이벤트 읽는 법

레이스가 진행되면 여러 정보가 동시에 갱신됩니다.

### 자주 보게 되는 이벤트

| 이벤트 | 화면 반응 |
|--------|-----------|
| **스킬 발동** | 플래시, 배너, 일부 경우 속도선 / 트레일 |
| **추월 발생** | 순위 변경, 로그 출력, 화면 강조 |
| **버스트 상태** | 주자 후방 시각 효과 강화 |
| **결승 통과** | 결승 연출, 완주 수 증가, 최종 순위 확정 |

이 이벤트들은 `activeSkillBanners`와 레이스 로그를 함께 소비하는 구조를 기준으로 점점 더 풍부하게 연결될 예정입니다.

### 관전 팁

- 우측 오버레이는 상위권 흐름을 빠르게 파악할 때 유용합니다.
- 미니맵은 전체 거리 대비 상대 위치를 확인하기 좋습니다.
- 로그는 어떤 스킬이 역전에 영향을 줬는지 읽을 때 가장 유용합니다.

## 8. 결과 화면 보기

모든 팀이 결승선을 통과하면 결과 화면으로 전환됩니다.

### 확인 가능한 정보

| 항목 | 설명 |
|------|------|
| **최종 순위** | 결승선 통과 순서 |
| **기록 요약** | 각 팀의 레이스 결과 정리 |
| **주요 이벤트** | 스킬 발동과 추월 흐름 요약 |
| **통계 정보** | MVP, 다수 발동, 베스트 추월 등 확장 가능 |

결과 화면은 단순 순위표가 아니라, 레이스 흐름을 되짚는 요약 보드 역할을 합니다.

## 9. 다시 달리기

레이스가 끝나면 다음 두 방식으로 재시작할 수 있습니다.

| 방식 | 설명 |
|------|------|
| **같은 팀으로 다시 달리기** | 팀 조합은 유지하고 결과만 다시 시뮬레이션 |
| **처음부터 다시 구성** | 참가 팀을 새로 편성 |

같은 팀 구성으로 다시 달려도 컨디션과 이벤트 흐름에 따라 결과는 달라질 수 있습니다.

## 10. 자주 묻는 질문

**Q. 왜 최대 12팀까지만 지원하나요?**  
A. 가독성과 연출 품질, 그리고 Pixi 씬 성능을 함께 고려한 현재 기준입니다.

**Q. 레이스 결과는 매번 같은가요?**  
A. 아닙니다. 컨디션, 스킬 발동, 추월 흐름 때문에 결과가 달라질 수 있습니다.

**Q. 미니맵과 실제 트랙 위치가 항상 같아야 하나요?**  
A. 네. 현재 구조와 고도화 계획 모두 `거리 → 트랙 좌표`, `거리 → 미니맵 좌표`의 일관성을 중요한 기준으로 봅니다.

**Q. 사운드는 이미 들어가 있나요?**  
A. 문서 작성 시점 기준으로 레이스 오디오 시스템은 고도화 단계에서 추가 예정입니다.

**Q. 왜 12팀 기준 성능을 따로 강조하나요?**  
A. 이 프로젝트의 핵심 목표 중 하나가 최대 참가 수에서도 프레임 드랍이 체감되지 않도록 유지하는 것이기 때문입니다.

**Q. 모바일에서도 바로 쓰기 좋은가요?**  
A. 현재는 PC 화면 중심 사용을 권장합니다.
